## 3D 세계  

## 피킹
> 오브젝트 및 기즈모 피킹 방법이다.
만약, line-line intersection, line-triangle intersection을 잘 모른다면 아래 사전학습을 숙지하는 것을 추천한다.

현재 클릭한 마우스 위치와 카메라를 기준으로 Ray 생성 
( Screen -> NDC -> View -> World)

### Screen To NDC
Screen [0, viewport) -> NDC [-1, 1] 가 되도록 변환

```cpp
FVector ndcPos;
ndcPos.X = (MouseX / viewportWidth) * 2.0f - 1.0f;
ndcPos.Y = 1.0f - (MouseY / viewportHeight) * 2.0f; // Screen의 Y는 아래로 갈 수록 증가, NDC의 Y는 위로 갈 수록 증가 
ndcPos.Z = 0.0f; // Near Plane
```

### NDC To View 
Inverse Perspective/Orthographic Project 곱하기 
```cpp
FMatrix invProjectionMatrix = FMatrix::Inverse(camera->GetProj());
FVector viewPos = invProjectionMatrix.TransformPointRow(ndcPos); // row vector 방식으로 행렬-벡터간 곱
```

### View To World 
Inverse View Matrix 곱하기 
```cpp
FMatrix invViewMatrix = FMatrix::Inverse(camera->GetView());
FVector worldPos = invViewMatrix.TransformPointRow(viewPos);
```

### Ray생성
```cpp 
if (camera->IsOrtho())
{
    FRay resultRay;
    resultRay.Origin = worldPos;
    resultRay.Direction = camera->GetForward();
    return resultRay;
}
FRay resultRay;
resultRay.Origin = camera->GetLocation(); // 카메라의 월드 위치
resultRay.Direction = (worldPos - resultRay.Origin).Normalized();
resultRay.MousePos = { MouseX, MouseY };
```

### Ray-Mesh 교차검증
좌표계를 우선 통일 시켜야 하기에, World Space로 통일했다. 
```cpp
TOptional<FVector> URaycastManager::RayIntersectsTriangle(FVector triangleVertices[3])
{
	constexpr float epsilon = std::numeric_limits<float>::epsilon();

	FVector edge1 = triangleVertices[1] - triangleVertices[0];
	FVector edge2 = triangleVertices[2] - triangleVertices[0];
	FVector ray_cross_e2 = RayDirection.Cross(edge2);
	float det = edge1.Dot(ray_cross_e2);

    // 평행 또는 일치 
	if (det > -epsilon && det < epsilon)
		return {};    

    // u 구하기 
	float inv_det = 1.0f / det;
	FVector s = RayOrigin - triangleVertices[0];
	float u = inv_det * s.Dot(ray_cross_e2);

    // u = [0, 1] 조건을 만족하지 못하면, 교차 X 
	if ((u < 0 && abs(u) > epsilon) || (u > 1 && abs(u - 1) > epsilon))
		return {};

    // v 구하기
	FVector s_cross_e1 = s.Cross(edge1);
	float v = inv_det * RayDirection.Dot(s_cross_e1);


    // v = [0, 1] 조건을 만족하지 못하면, 교차 X 
	if ((v < 0 && abs(v) > epsilon) || (u + v > 1 && abs(u + v - 1) > epsilon))
		return {};

    // t 구하기 
	float t = inv_det * edge2.Dot(s_cross_e1);

	if (t > epsilon) // ray intersection
	{
		return FVector(RayOrigin + RayDirection * t);
	}
    // ray의 뒷쪽에 존재하기 때문에, intersect하지 않는다. 
	else 
		return {};
}
```

# Picking을 이해하기 위한 사전자료

# 1. 선과 선의 교차 (Line-Line Intersection)

### 1-1. 직선의 표현
---

무한한 길이의 직선은 서로 다른 두 점으로 표현한다.

> L1: (x1, y1), (x2, y2)  
> L2: (x3, y3), (x4, y4)

방향 벡터는 다음과 같다.

> v1 = (x2 - x1, y2 - y1)  
> v2 = (x4 - x3, y4 - y3)


### 1-2. 선과 선의 관계 (3가지)
---

두 직선은 다음 세 가지 중 하나이다.

1) 한 점에서 교차  
2) 서로 평행  
3) 완전히 일치  

이를 판단하는 핵심이 determinant(행렬식)이다.

### 1-3. determinant의 기하학적 의미
---

2차원 벡터  
a = (ax, ay)  
b = (bx, by)

det(a, b) = ax * by - ay * bx

> det = 0 → 두 벡터는 평행  
> det > 0 → b는 a 기준 반시계 방향  
> det < 0 → b는 a 기준 시계 방향

즉, determinant는 평행 여부 + 방향 + 면적 정보를 가진다.


### 1-4. 교차점 계산 (Cramer's Rule)
---

직선의 매개변수 표현:

L1: P1 + t * v1  
L2: P3 + s * v2  

교차 조건:

P1 + t * v1 = P3 + s * v2

이를 2×2 선형방정식으로 정리하면 크래이머 공식으로 풀 수 있다.   
(크래이머 공식은 따로 찾아보시길...)

크래이머 공식을 사용하면 분모는 det(v1, v2)가 된다. 
> denominator != 0 -> 유일한 교차점 존재  
> denominator = 0 -> 평행 또는 일치
 

# 2. 선과 삼각형의 교차 (Ray–Triangle Intersection)

### 2-1. 기본 지식
---
Picking을 하기 위해서는 
삼각형 ABC와 광선(ray)이 주어졌을 때,  
광선이 삼각형과 교차하는지 판별하고, 교차점 P를 구할 수 있어야 한다.

광선(Ray) 표현식
> R(t) = O + tD  
> O: Ray Origin  
> D: Ray Direction  

삼각형 표현식
> A, B, C (삼각형의 세 정점)


목표
1) 교차 여부 판정  
2) 교차점 P 계산  
3) 삼각형 내부인지 판정  

## 2-2. 기본 아이디어
삼각형 내부의 점 P는 baricentric coordinate로 표현 가능하다.

> P = (1 - u - v)A + uB + vC  
> 조건: u,v = [0,1], u + v ≤ 1

Ray 위의 점은
> P = O + tD

따라서 교차 조건은 아래와 같다.  
>O + tD = A + u(B - A) + v(C - A)

## 2-3. Möller–Trumbore 알고리즘

삼각형의 edge 정의:

> e1 = B - A  
> e2 = C - A  

Ray와 삼각형 (Baricentric Coordinate) 교차 식
>O + tD = A + u e1 + v e2

정리

>O - A = -tD + u e1 + v e2


이를 3×3 선형 시스템으로 보면:

(O - A) = [e1 e2 -D] * (u v t)^T

이를 Cramer's Rule로 풀면 아래와 같이 된다.

### Step 1: Edge 계산

e1 = B - A  
e2 = C - A  

---

### Step 2: 평행성 검사

pvec = D × e2  
det = e1 · pvec  

> det ≈ 0 → Ray와 삼각형 평면이 평행  
> → 교차 없음

---

### Step 3: u 계산

tvec = O - A  
u = (tvec · pvec) / det  

> u < 0 또는 u > 1 → 삼각형 밖

---

### Step 4: v 계산

qvec = tvec × e1  
v = (D · qvec) / det  

> v < 0 또는 u + v > 1 → 삼각형 밖

---

### Step 5: t 계산

t = (e2 · qvec) / det  

> t < 0 → 광선 반대 방향  
> t ≥ 0 → 교차점 존재

---

## 2-5. 최종 교차점

> P = O + tD

---

## 2-6. det의 기하학적 의미 

det = e1 · (D × e2)

이는 3차원 scalar triple product이다.

|det| = 평행육면체 부피 (이건 식을 풀어서 보면 알 수 있는데, 여기서는 패스하겠다)  
→ det = 0 이면  
Ray 방향 D, 삼각형 edge e1, e2가 같은 평면에 있음  
→ 평행

---

### u, v의 의미

u, v는 바리센트릭 좌표이다.

> P = A + u e1 + v e2  
> = (1 - u - v)A + uB + vC

조건:

u ≥ 0  
v ≥ 0  
u + v ≤ 1  

→ 삼각형 내부 판정 조건

---

## 2-7. 장점

- 평면 방정식 계산 불필요
- 빠른 연산 (외적 2번, 내적 몇 번)
- 분기 최소화 가능
- Ray tracing에서 표준 사용

---

## 2-8. C++ 의사 코드

```cpp
bool RayTriangle(
    Vec3 O, Vec3 D,
    Vec3 A, Vec3 B, Vec3 C,
    float& t)
{
    Vec3 e1 = B - A;
    Vec3 e2 = C - A;

    Vec3 pvec = cross(D, e2);
    float det = dot(e1, pvec);

    if (abs(det) < EPS)
        return false;

    float invDet = 1.0f / det;

    Vec3 tvec = O - A;
    float u = dot(tvec, pvec) * invDet;
    if (u < 0 || u > 1)
        return false;

    Vec3 qvec = cross(tvec, e1);
    float v = dot(D, qvec) * invDet;
    if (v < 0 || u + v > 1)
        return false;

    t = dot(e2, qvec) * invDet;
    return t >= 0;
}
