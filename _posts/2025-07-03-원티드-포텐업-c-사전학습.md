### Null pointer exception을 봐보자

nullptr에 접근하려는 것은 사실 0번지 주소에 접근하려고하는 것

근데 왜 0번지에 접근하는게 오류일까??

메모리를 할당하는 과정으로 예를 들면

메모리 할당을 os에게 부탁, os는 cpu한테 부탁

cpu는 0번지에 접근하려는 명령어를 보고, 거기는 os자리야! 라면서 interrupt를 걸어서 취소시킴

(0번지는 항상 os 자리이다. + 요즘에는 운영체제 단에서 software interrupt를 걸어서 처리한다.)

[CRT 라이브러리로 메모리 누수 찾기

C/C++ 디버거 및 CRT(C 런타임 라이브러리)를 통해 메모리 누수를 찾는 방법을 알아봅니다. 포함되는 기술로는 메모리 누수 보고서와 메모리 스냅샷 비교가 있습니다.

learn.microsoft.com](https://learn.microsoft.com/ko-kr/cpp/c-runtime-library/find-memory-leaks-using-the-crt-library?view=msvc-170)

해당 문서를 따라서

```
#define _CRTDBG_MAP_ALLOC
#include <stdlib.h>
#include <crtdbg.h>
```

이 부분만 추가하면 아래와 같이 누수를 체크할 수 있다.

![](https://blog.kakaocdn.net/dna/bLeUfa/btsO2aGplfH/AAAAAAAAAAAAAAAAAAAAACjzjSZjVrRuW-siV4gyX2Gs3ZHTscHe804MGI2dJC6E/img.png?credential=yqXZFxpELC7KVnFOS48ylbz2pIh7yKj8&expires=1772290799&allow_ip=&allow_referer=&signature=6%2BYtqWoaa40xuJTjNk9Bebcb1J0%3D)

정확하게 알려주지만 위치를 안알려준다.

아래 define문을 추가하자

```
#ifdef _DEBUG
    #define DBG_NEW new ( _NORMAL_BLOCK , __FILE__ , __LINE__ )
    // Replace _NORMAL_BLOCK with _CLIENT_BLOCK if you want the
    // allocations to be of _CLIENT_BLOCK type
#else
    #define DBG_NEW new
#endif
```

![](https://blog.kakaocdn.net/dna/2IhUe/btsO0EuFGCe/AAAAAAAAAAAAAAAAAAAAAMulT2ymKK-BZzL2bzCE0-IrLapyHxWD5daMr4QdgQ4U/img.png?credential=yqXZFxpELC7KVnFOS48ylbz2pIh7yKj8&expires=1772290799&allow_ip=&allow_referer=&signature=5BVzDeAgMdry%2F290A20pg1qMKoI%3D)

이제 메모리 누수 위치를 36번째 줄이라고 알려준다. (더블 클릭하면 해당 위치로 이동까지 해준다.)

이게 어떻게 가능하냐?

#define에서 NORMAL\_BLOCK 어쩌구, FILE어쩌구, LINE어쩌구하면서 부가적인 정보를 기입하는 것이다.

그럼 또 느려지겠지? 그러니까 Debug에서만 사용하는 것을 볼 수 있다.
